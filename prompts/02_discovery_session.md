# 보조 프롬프트 -- 공격 표면 탐색 실험 세션

개별 기법의 실현 가능성이 불확실할 때, 하네스 루프 밖에서 빠르게 실험하고 돌아온다.

---

```
## 실험 세션 시작: 공격 표면 탐색

현재 목표: [실험할 AWS 서비스 조합 — 예: "EKS Pod에서 IMDS를 통한 노드 IAM Role 탈취"]
AWS 계정 ID: [계정 ID]
리전: us-east-1
현재 단계: 공격 표면 탐색 (Attack Surface Discovery)

### 이전 실험 결과 요약
[이전 세션에서 확인된 사실 — 없으면 "첫 실험"]

### 이번 세션의 실험 목표
실험하고 싶은 공격 벡터:

1. [기법 1 — 예: "Lambda Layer를 교체해서 다른 Lambda가 로드할 때 환경변수를 탈취할 수 있는가?"]
2. [기법 2 — 예: "CloudFormation CreateStack + PassRole로 Admin 역할을 생성할 수 있는가?"]
3. [기법 3 — 예: "EventBridge PutRule로 기존 이벤트를 내 Lambda로 라우팅할 수 있는가?"]

### 각 실험에 대해 해줄 것

실험마다 다음 순서로 진행해:

**Step 1 — 가설 작성**
"어떤 권한이 있으면 어떤 결과가 나올 것이다"를 명확히 써줘

**Step 2 — 최소 Terraform 작성**
가설을 검증하기 위한 최소한의 인프라를 Terraform으로 작성해줘.
모든 리소스에 태그 달고, variable로 재사용 가능하게 만들어줘.

**Step 3 — 배포 & 실행**
terraform apply 후 실제 공격 명령어를 실행해줘.
모든 명령어의 전체 출력(에러 포함)을 기록해줘.

**Step 4 — 결과 분석**
- 성공했으면: 최소 필요 권한, 필요한 조건, 방어 방법을 정리
- 실패했으면: 에러 메시지를 분석하고, "왜 실패했는가?" + "우회 가능한가?" 분석

**Step 5 — 발견 기록**
이 실험에서 새로 알게 된 사실을 정리. 특히:
- AWS 문서에 명시되지 않은 동작
- 예상과 다른 결과
- 방어자 관점에서의 시사점

**Step 6 — 다음 실험 제안**
이 결과를 바탕으로 다음에 해볼 만한 실험을 제안

### 제약 조건
- 시간 제한: 2시간
- 비용 제한: $5 이내
- 실험 후 반드시 terraform destroy

실험을 시작해줘. 첫 번째 기법의 가설 수립부터 시작해.
```

현재 목표: Lambda Layer Injection을 통한 다른 Lambda의 Secret 탈취
AWS 계정 ID: 542551366766
리전: us-east-1
현재 단계: 공격 표면 탐색

### 이전 실험 결과 요약

- Lambda Layer는 아직 실험하지 않음

- 시간: 1시간
