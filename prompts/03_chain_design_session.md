# 보조 프롬프트 -- 공격 체인 초안 설계

> 주의: 이 프롬프트로 만드는 것은 **초안 스케치**다. 상세 설계가 아니다.
> 체인의 큰 그림만 잡고, 각 Stage의 실제 검증은 07_harness_loop.md에서 한다.
> 초안 설계에 30분 이상 쓰지 마라. 빨리 스케치하고 바로 Stage 0 구현으로 넘어가라.
> 루프를 돌면서 체인은 계속 바뀐다. 처음부터 완벽할 필요 없다.

---

```
## 공격 체인 초안 설계

시나리오 코드네임: [이름 -- 예: "invisible_pipeline"]
목표 난이도: Expert

### 실험으로 검증된 기법 목록

탐색 실험에서 확인된 공격 기법을 아래에 정리해줘:

| # | 기법 | 실험 ID | 필요 권한 | 성공 조건 | 비고 |
|---|------|---------|-----------|-----------|------|
| 1 | [기법명] | [EXP-001] | [IAM 권한] | [조건] | [특이사항] |
| 2 | ... | ... | ... | ... | ... |

### 시나리오 요구사항

- **진입점**: [SSRF / RCE / SQL Injection / Auth Bypass 중 택 1]
- **최소 피봇 단계**: 5
- **필수 포함 요소**:
  - [ ] 래터럴 무브먼트 (다른 서비스/역할로 이동)
  - [ ] 크레덴셜 탈취 (최소 2종류의 자격증명 형태)
  - [ ] 권한 상승 (낮은 권한 → 높은 권한)
  - [ ] 네트워크 경계 넘기 (VPC/서브넷 간 이동)
- **미끼 경로**: 최소 2개 (핵심 경로보다 먼저 눈에 띄어야 함)
- **Flag 위치**: 최종 단계에서만 접근 가능한 곳

### 해줄 것 (3개만. 나머지는 루프 돌면서 나온다)

**1. 공격 체인 다이어그램**
ASCII art로 전체 공격 흐름을 그려줘. 각 Stage마다:
- 사용 기법 (한 줄)
- 획득하는 것 (한 줄)
- 다음 Stage로의 연결 (한 줄)
- 미끼가 어디서 갈라지는지 표시

**2. 불확실한 단계 표시**
각 Stage에 대해 "실험으로 검증됨 / 아직 미검증" 표시.
미검증 단계는 하네스 루프에서 단독 검증할 때 실패할 수 있다는 걸 인지하고 시작.

**3. 신규성 검증 (필수)**
REFERENCE.md의 "신규성 검증" 섹션에 있는 기존 시나리오 DB와 비교해서:
- 가장 유사한 기존 시나리오 3개를 찾고
- 4가지 비교 항목(진입점/피봇경로/권한상승/최종도달)에서 몇 개가 다른지 판정
- 3개 이상 다르면 NOVEL -> 진행
- 2개 이하면 체인을 수정한 뒤 다시 판정
- 결과를 harness_state.md의 "신규성 판정" 섹션에 기록

IAM 매트릭스, 네트워크 토폴로지, 난이도 평가는 루프를 돌면서 자연스럽게 나온다. 지금은 만들지 마라.

### 설계 제약
- 시간당 운영 비용 $2 이하
- 배포 시간 10분 이내 (대형 서비스 제외)
- terraform destroy 한 번으로 완전 정리
- 리전 단일 (us-east-1)
```

## 초안 완성 후: 하네스 루프로 이동

초안이 나오면 07_harness_loop.md의 "첫 세션용" 변형을 사용해서 Stage 0부터 구현/검증을 시작한다.
아래 프롬프트는 루프를 돌면서 체인 전체를 재검토할 때 사용한다:

```
## 공격 체인 검증

시나리오: [코드네임]
설계된 체인: [Stage 0→1→2→3→4→5 요약]

### 검증할 것

각 Stage를 순서대로 실험해줘. Stage N의 결과물이 Stage N+1의 입력이 되는지 확인:

1. **Stage 0 → 1 연결**: 웹 취약점에서 얻은 것으로 정찰이 가능한가?
2. **Stage 1 → 2 연결**: 정찰 결과로 래터럴 무브먼트가 가능한가?
3. **Stage 2 → 3 연결**: 이동한 위치에서 크레덴셜을 탈취할 수 있는가?
4. **Stage 3 → 4 연결**: 탈취한 크레덴셜로 권한 상승이 가능한가?
5. **Stage 4 → 5 연결**: 상승된 권한으로 Flag에 접근할 수 있는가?

각 연결 지점에서:
- 성공: 다음 Stage로 진행
- 실패: 실패 원인 분석 + 체인 수정 제안

미끼 경로도 테스트:
- 미끼 1: 풀이자가 이 경로를 시도하면 어디서 막히는가?
- 미끼 2: 위와 동일

### 결과물
- 검증 완료된 최종 공격 체인 다이어그램
- 각 Stage의 정확한 명령어 목록
- 체인에서 수정이 필요한 부분 목록
```
