# /compact 보존 지침 템플릿

컨텍스트 60%에 도달하면 아래를 복사해서 `/compact` 뒤에 붙여넣는다.
`[...]` 부분만 현재 상태로 채워라.

## 멀티에이전트 모드 (/orchestrate 사용 시)

```
/compact Keep the following:

## 시나리오 상태
- 코드네임: [시나리오명]
- 공격 체인: [Stage 0: 기법] -> [Stage 1: 기법] -> ... -> [Stage N: 기법]
- 현재 작업 Stage: Stage [N]
- 검증 상태: Stage [0~M] 검증 완료, Stage [N] 작업 중

## 멀티에이전트 상태
- 현재 조율 패턴: [A/B/C/D/E/F]
- 마지막 에이전트 작업 결과:
  - Terraform Engineer: [결과 요약]
  - Attack Executor: [결과 요약]
  - 기타: [결과 요약]
- 다음에 스폰할 에이전트: [에이전트명] + [전달할 컨텍스트]

## 이번 세션 진행 상황
- 완료한 것: [어떤 검증/구현을 끝냈는지]
- 현재 상태: [지금 정확히 뭘 하고 있는지]
- 미해결 이슈: [에러 메시지, 실패 원인, 막힌 것]

## 핵심 발견
- [이번 세션에서 새로 알게 된 것 -- AWS 동작, 에러 패턴 등]

## Terraform 상태
- 마지막 apply 결과: [성공/실패]
- 변경했지만 아직 apply 안 한 것: [있으면 기록]

## 다음 단계
- [compact 후 바로 이어서 할 것]
- [어떤 에이전트를 어떤 패턴으로 스폰할지]

## 변하지 않는 규칙 (항상 유지)
- 멀티에이전트: /orchestrate로 에이전트 조율
- 에이전트 프롬프트: .claude/agents/ 디렉토리
- 3단계 검증: 단독 -> 연결 -> 회귀
- 실패 시 에러 전문 기록 -> 수정 -> 재검증
- 검증 성공 즉시 validation/ 스크립트 생성
```
