# 실패/발견 심화 분석

실패 상황: $ARGUMENTS

## 실패 심화 분석

1. **에러 원인 분석**: 정확히 왜 발생하는가?
   - IAM 권한 부족? -> 어떤 권한 필요?
   - 리소스 조건? -> 어떤 조건 필요?
   - 네트워크 제한? -> 어떤 경로 차단?
   - 서비스 자체 제한? -> 우회 가능?

2. **우회 가능성 탐색**:
   - 다른 API 엔드포인트?
   - 다른 서비스 경유?
   - 시간/순서 변경?
   - 조건 변경? (태그 변경, 설정 수정)

3. **시나리오 활용**: 이 실패를 난이도 요소/미끼/데드엔드로 활용 가능?

4. **추가 실험 제안**: 우회 경로, 조건 변경, 관련 서비스 조합

## 예상치 못한 성공 심화

1. **왜 가능한가?** (AWS 문서에 명시? 의도된 동작? 버그?)
2. **공격 활용**: 어떤 자격증명/정보 획득 가능? 영향 범위? CloudTrail 탐지?
3. **재현 가능?**: 최소 조건 정리, Terraform 자동 배포, 타이밍 이슈
4. **시나리오 통합**: 어떤 Stage에 배치? 힌트 설계?

## 서비스 간 상호작용 탐색

두 서비스 상호작용 시:
1. A의 자격증명으로 B 제어 가능?
2. B가 A 호출 시 어떤 자격증명 사용?
3. 상호작용 가로채기 가능?
4. 민감 정보 노출?
