# 하네스 루프 핵심 규칙

이 규칙은 모든 세션에 항상 적용된다.

## 작업 규칙

1. **한 조각씩 만들고 즉시 검증**: Terraform 리소스 1-2개 추가 -> apply -> 공격 테스트. 큰 덩어리를 한 번에 만들지 마라
2. **3단계 검증 매번 수행**: 단독 -> 연결 -> 회귀. 하나라도 실패하면 다음 Stage로 안 넘어간다
3. **검증 성공 즉시 스크립트화**: 수동 검증에 성공한 명령어를 `validation/test_stage_N.sh`로 만들어라
4. **실패 -> 수정 -> 재검증 루프**: 에러 전문 기록 -> 원인 분류(IAM/네트워크/타이밍/명령어) -> 수정 -> 재실행
5. **추측하지 마라**: "~할 수 있을 것이다"가 아니라 "실험으로 확인했다"만 허용
6. **Terraform 코드는 idempotent**: `destroy -> apply`가 반복 가능해야 한다
7. **매 세션 끝에 harness_state.md 업데이트**: 어디까지 검증됐는지 추적

## 검증 결과 기록 형식

```
### 검증 결과: Stage [N]

#### 단독 검증
- 명령어: [실행한 것]
- 결과: PASS/FAIL
- 출력: [전문]

#### 연결 검증 (Stage N-1 -> N)
- 입력: [이전 Stage에서 가져온 것]
- 결과: PASS/FAIL

#### 회귀 검증 (Stage 0 -> N)
- terraform destroy + apply: PASS/FAIL
- 각 Stage 결과: [기록]
- 총 소요시간: [분]
```
