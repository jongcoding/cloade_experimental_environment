# 컨텍스트 관리 규칙

이 규칙은 모든 세션에 항상 적용된다.

## /compact 사용 규칙

1. **60% 규칙**: 컨텍스트 사용량이 60%에 도달하면 선제적으로 `/compact`를 실행해라. 경고가 뜰 때까지 기다리지 마라 -- 그때는 이미 품질이 떨어진 상태다
2. **보존 지침 필수**: `/compact`를 실행할 때 반드시 보존할 내용을 명시해라. 맨 `/compact`만 치지 마라
3. **압축 후 검증**: `/compact` 직후 "현재 상태와 다음 작업을 요약해줘"를 실행해서 누락 확인
4. **반복 가능**: 한 세션에서 여러 번 `/compact` 가능. 매번 60% 규칙 적용

## /compact 보존 지침 템플릿

```
/compact Keep the following:
1. 현재 시나리오 코드네임과 전체 공격 체인 구조
2. harness_state.md의 현재 상태 (어떤 Stage까지 검증됐는지)
3. 이번 세션에서 발견한 것 (실패 원인, 새 발견, 미해결 이슈)
4. 현재 작업 중인 Stage와 정확한 에러/상황
5. 아직 시도하지 않은 다음 단계
6. Terraform 변경 사항 중 아직 apply 안 한 것
```

## 세션 상태 저장

1. **harness_state.md는 무조건 업데이트**: 세션 중간이라도 중요한 진전이 있으면 harness_state.md에 기록. 세션 끝에만 하지 마라
2. **실패 즉시 기록**: 에러 메시지 전문을 experiment_log/에 저장. 컨텍스트에서 밀려나도 파일로 남아야 한다
3. **검증 스크립트는 즉시 생성**: 수동 검증 성공 시 validation/ 스크립트로 만들어두면 컨텍스트를 잃어도 재현 가능

## Auto-Memory 활용

Claude Code의 Auto-Memory가 활성화되어 있으면 세션 간 학습이 자동으로 유지된다.
수동으로 기억시키고 싶은 것이 있으면 `/memory` 명령어로 관리할 수 있다.

## 긴 세션 전략

1. **세션 시작**: harness_state.md + CLAUDE.md 읽기로 컨텍스트 복원
2. **작업 중**: 60% 도달 시 `/compact` + 보존 지침
3. **작업 중간**: 중요 발견/변경 시 harness_state.md 중간 업데이트
4. **세션 종료**: harness_state.md 최종 업데이트 + 다음 세션 할 것 명시
