# 멀티에이전트 강제 규칙

이 규칙은 `/orchestrate` 스킬 사용 시 항상 적용된다.

## 절대 규칙

1. **오케스트레이터는 직접 실행하지 않는다**:
   - `terraform` 명령어를 직접 실행하지 마라
   - `aws` CLI 명령어를 직접 실행하지 마라
   - `kubectl` 명령어를 직접 실행하지 마라
   - `bash validation/*` 스크립트를 직접 실행하지 마라
   - 위 작업은 **반드시 Agent 도구로 서브에이전트를 스폰해서 위임**해라

2. **오케스트레이터가 직접 해도 되는 것**:
   - 파일 읽기 (harness_state.md, REFERENCE.md, Terraform 코드 등)
   - 파일 쓰기 (harness_state.md 업데이트)
   - 에이전트 간 데이터 전달 판단
   - 다음 에이전트 스폰 결정
   - `/compact` 실행 판단
   - 사용자와 대화

3. **에이전트 스폰 시 필수 포함 정보**:
   - 해당 에이전트의 프롬프트 파일(`.claude/agents/[name].md`) 내용
   - 시나리오 코드네임
   - 현재 작업 디렉토리 경로
   - harness_state 현재 상태 (검증 테이블)
   - 이전 에이전트의 결과 (있으면)
   - 구체적 작업 지시

4. **병렬 스폰 의무**:
   - 독립적인 작업은 반드시 하나의 메시지에 여러 Agent 호출로 병렬 스폰해라
   - 순차가 필요한 경우에만 순차로 스폰해라
   - "일단 하나 먼저 해보고" 식의 불필요한 순차 실행 금지

## 위반 시

오케스트레이터가 직접 `terraform apply`나 `aws` 명령어를 실행하려고 하면,
그것은 싱글에이전트 모드(`/harness-loop`)로 빠진 것이다.
멀티에이전트 모드에서는 **무조건 Agent 도구를 통해 위임**해라.
